## 5. Session_ID 결측 패턴

Session_ID가 결측인 241,502건을 Event_Type별로 분류하면:

| Event_Type | 결측 수 |

| 챌린지참여 | 1,454 |
| 챌린지_탐색 | 1,191 |
| 온보딩_완료 | 75 |

**해석:**  
- `알림수신` + `알림오픈` = 215,543건: 푸시 알림은 앱 세션 없이 발생 → **구조적 결측 (정상)** 🟢  
- `앱실행` 10,610건: 세션 시작 직전 로그일 가능성. **경미한 이슈** 🟡  
- 나머지 행동 이벤트 9,718건: 이유 불명 → **이슈** 🔴  

**권장 처리 방향 (요상의):** Session 기반 분석 시 알림 이벤트는 제외하고, 나머지 결측 Session_ID는 `앱실행` 직후 임시 세션 할당 여부를 별도로 논의.
